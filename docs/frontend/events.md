# 事件綁定政策（避免衝突與重複綁定）

目標：在 Partial/Modal 動態載入的情境下，確保事件可靠綁定、不重複、不受重繪影響。

## 基線原則
- 預設由按鈕 `onclick` 觸發函式，保持語意直接、易讀
- 需要動態內容（Partial/Modal 反覆載入）時，再使用 jQuery「委派事件」：`$(document).on('click', '.selector', handler)`
- 使用事件命名空間（例如 `.form`, `.nav`）避免互相解除時誤傷（適用於委派事件）
- 在綁定前加上 `.off('event.namespace', selector)` 再 `.on(...)`，避免重複綁定（適用於委派事件）

```javascript
function onSave(){ /* ... */ }
$(document)
  .off('click.form', '.js-save')
  .on('click.form',  '.js-save', onSave);
```

## 遷移策略（避免「一刀切」）
- 不做「全域清洗元素端綁定」的硬性限制：先盤點後遷移
- 遷移順序：
  1) 點擊行為、穩定 DOM 使用按鈕 `onclick`；需要動態時再改委派
  2) 改為委派時，將元素端事件屬性移除，改以 class 選擇器配合委派
  3) 發佈與驗證；確認功能無回歸后，再進行下一批

## 動態內容與 Modal
- Modal/Partial 的內容多由 AJAX 動態載入，使用委派事件可避免「載入後需再次 bind」的問題
- 僅在極少數封裝元件（例如單檔小控件）可使用 `addEventListener`，但需保證：
  - 事件綁定只發生一次
  - 卸載/銷毀時能解除綁定

## 共同錯誤模式（避免）
- 在 `.ready()` 內對動態元素直接 `.on('click', '#id', ...)`（ID 綁住易衝突，且非委派）
- 同元素多處綁定無命名空間，導致解除事件時誤解除
- 同一元素同時以多種綁定策略重複綁定相同行為，造成重複執行

## 驗證清單
- 是否使用 `.off(...).on(...)` 搭配命名空間
- 是否避免同一元素重複綁定（新程式碼）
- 是否避免對將被 AJAX 置換的節點做直接綁定
- 是否在 Scripts Partial 統一定義頁面事件
