# WW 介面風格說明（工作台式產品 UI）

## 核心定位
本介面採用「工作台（Workbench / 操作台）」風格：把使用者要完成的任務拆成「可控輸入 → 系統產出 → 可調整 → 可追溯」的閉環，並在同一個穩定骨架下完成，而非以對話串作為主要承載。

## 為何不是典型 AI Chat 風格
- Chat 風格會把條件、狀態、結果分散在訊息流中，回看與調整成本高。
- 工作台風格把「條件」固定在可視控制區，把「結果/狀態」放在畫布區，讓使用者感覺可控、可預期，並適合高頻重複操作。

## 版型骨架（固定不變的框架）
- 左側窄側欄：穩定的全域導覽（模組切換），避免在內容區反覆換頁迷路。
- 上方 Topbar：顯示當前情境（標題/位置/日期/身份），提供「我在哪裡」的定位感。
- 主要區域採「控制面板 + 主畫布」思維：
  - 控制面板：集中放條件、篩選、輸入、主要操作按鈕。
  - 主畫布：顯示狀態、結果、細節，並支援同頁模式切換（例如空狀態/載入/結果/詳情）。

## 資訊呈現策略
- 高資訊密度但不混亂：以卡片、分區、細線框與圓角做模組化切割。
- 先總結再下鑽：先給摘要或聚合卡，再進一步查看細節，避免長清單造成無限捲動。
- 狀態機導向：用明確的空狀態、進行中、完成、待確認等狀態呈現，讓流程可追蹤。

## 互動策略（可控、快速、可回退）
- 以「選擇/篩選/短輸入」為主，避免長對話輸入成為主要操作。
- 重要操作集中成主 CTA（例如生成/提取/保存），次要操作以小按鈕或 icon button 放在結果附近。
- 任何 AI 相關功能以「輔助填表/輔助決策」呈現：
  - 使用者提供可控條件
  - 系統給出可檢視理由的產出
  - 允許微調、重做、保存、回看

## 視覺語言（溫暖、柔和、工具感）
- 配色以米色系底（溫暖中性）+ 橘色作為品牌/主操作強調色，避免冰冷企業感。
- 大量圓角（約 9–14px）與低對比細邊框（0.5px/1px）形成「柔和的系統 UI」。
- 字級偏小、行距克制，強調效率與同屏承載能力。
- icon 使用線性圖示（Lucide），作為語意提示而非裝飾；避免「檔案總管/資料夾」意象造成誤解。

## 內容與敘事風格
- 文案偏「系統解釋」：短、可驗證、偏原因/建議，而非長篇聊天。
- 結果區同時包含「產出」與「下一步行動」，讓使用者不用思考下一步該做什麼。

## 設計原則（可延伸到新畫面）
- 固定骨架不變：側欄/Topbar/控制面板/畫布保持一致，新增功能只是在畫布中新增模式或卡片。
- 新增功能優先做「可控輸入 + 可追溯結果」：
  - 可控：條件集中、可見、可修改
  - 可追溯：記錄可回看、可下載、可重做、可連回來源物件
- 避免無限捲動：資料多時採「聚合卡 + 下鑽」或「分段載入」，不要把所有事件攤平成長 list。

## 典型元件語彙（建議沿用）
- 卡片（Card）、標籤/膠囊（Pill/Chip）、空狀態（Empty State）、載入狀態（Loading）、摘要（Summary）、詳情（Detail Panel）、批次/操作卡（Batch/Action Card）
