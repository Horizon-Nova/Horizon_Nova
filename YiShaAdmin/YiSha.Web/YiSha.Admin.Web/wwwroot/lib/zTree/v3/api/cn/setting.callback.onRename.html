Function(event, treeId, treeNode, isCancel)setting.callback.onRename

概述[ 依赖 jquery.ztree.exedit 扩展 js ]

用於捕獲節點編輯名稱結束之後的事件回調函數。

1、節點進入編輯名稱狀態，並且修改節點名稱後触發此回調函數。如果使用者設置了 beforeRename 回調函數，並返回 false，將无法触發 onRename 事件回調函數。

2、如果通過直接修改 treeNode 的資料，並且利用 updateNode 方法更新，是不会触發此回調函數的。

3、從 v3.5.13 開始，取消編輯狀態也会触發此回調，根据 isCancel 參數判斷

默認值：null

Function 參數說明

eventjs event 對象

標准的 js event 對象

treeIdString

對應 zTree 的 treeId，便於使用者操控

treeNodeJSON

被修改名稱的節點 JSON 資料對象

isCancelBoolean

是否取消操作 (v3.5.13+)

isCancel = true 表示取消編輯操作（按下 ESC 或 使用 cancelEditName 方法）

isCancel = false 表示確認修改操作

setting & function 举例

1. 修改名稱後，弹出被修改名稱的節點的 tId 以及 name 信息

function zTreeOnRename(event, treeId, treeNode, isCancel) {
	alert(treeNode.tId + ", " + treeNode.name);
}
var setting = {
	callback: {
		onRename: zTreeOnRename
	}
};
