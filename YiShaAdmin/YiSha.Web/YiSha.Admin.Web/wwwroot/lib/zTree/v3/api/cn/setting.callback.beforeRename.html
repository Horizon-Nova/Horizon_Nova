Function(treeId, treeNode, newName, isCancel)setting.callback.beforeRename

概述[ 依赖 jquery.ztree.exedit 扩展 js ]

用於捕獲節點編輯名稱結束（Input 失去焦點 或 按下 Enter 鍵）之後，更新節點名稱資料之前的事件回調函數，並且根据返回值確定是否允許更改名稱的操作

節點進入編輯名稱狀態後，按 ESC 鍵可以放弃當前修改，恢复原名稱，取消編輯名稱狀態

從 v3.5.13 開始，取消編輯狀態也会触發此回調，根据 isCancel 參數判斷

默認值：null

Function 參數說明

treeIdString

對應 zTree 的 treeId，便於使用者操控

treeNodeJSON

將要更改名稱的節點 JSON 資料對象

newNameString

修改後的新名稱

isCancelBoolean

是否取消操作 (v3.5.13+)

isCancel = true 表示取消編輯操作（按下 ESC 或 使用 cancelEditName 方法）

isCancel = false 表示確認修改操作

返回值Boolean

返回值是 true / false

如果返回 false，zTree 將保持名稱編輯狀態，无法触發 onRename 事件回調函數，並且会導致屏蔽其它事件，直到修改名稱使得 beforeRename 返回 true

如果返回 false，不会讓 input 輸入框獲取焦點，避免由於警告信息而導致反复触發 beforeRename。 請在關閉提示警告信息後，利用 editName 方法讓 input 重新獲取焦點。

setting & function 举例

1. 禁止修改的名稱的长度小於 5

function zTreeBeforeRename(treeId, treeNode, newName, isCancel) {
	return newName.length > 5;
}
var setting = {
	edit: {
		enable: true
	},
	callback: {
		beforeRename: zTreeBeforeRename
	}
};
